# Module_W25Q64 外部 Flash 存储模块文档

## 1. 概述

适用于 STM32 系列 MCU 的 W25Q64（8MB）外部串行 Flash 驱动模块。在高性能平台（如 STM32H7）上支持 **OCTOSPI (Quad-SPI 1-4-4)** 通信模式，提供高速的数据读写能力与内存映射功能。

---

## 2. 架构设计
### 2.2 状态与宏定义
```c
#define W25Qxx_CMD_EnableReset    0x66 // 使能复位
#define W25Qxx_CMD_ResetDevice    0x99 // 复位器件
#define W25Qxx_CMD_JedecID        0x9F // JEDEC ID
#define W25Qxx_CMD_WriteEnable    0X06 // 写使能

#define W25Qxx_CMD_SectorErase    0x20 // 扇区擦除，4K字节， 参考擦除时间 45ms
#define W25Qxx_CMD_BlockErase_32K 0x52 // 块擦除，  32K字节，参考擦除时间 120ms
#define W25Qxx_CMD_BlockErase_64K 0xD8 // 块擦除，  64K字节，参考擦除时间 150ms
#define W25Qxx_CMD_ChipErase      0xC7 // 整片擦除，参考擦除时间 20S

#define W25Qxx_CMD_QuadInputPageProgram  0x32 // 1-1-4模式下(1线指令1线地址4线数据)，页编程指令，参考写入时间 0.4ms
#define W25Qxx_CMD_FastReadQuad_IO       0xEB // 1-4-4模式下(1线指令4线地址4线数据)，快速读取指令
#define W25Qxx_CMD_ReadStatus_REG1       0X05 // 读状态寄存器1
#define W25Qxx_Status_REG1_BUSY          0x01 // 读状态寄存器1的第0位（只读），Busy标志位，当正在擦除/写入数据/写命令时会被置1
#define W25Qxx_Status_REG1_WEL           0x02 // 读状态寄存器1的第1位（只读），WEL写使能标志位，该标志位为1时，代表可以进行写操作
```


| 宏定义 | 值 | 说明 |
| --- | --- | --- |
| `W25Qxx_PageSize` | 256 | 页大小（编程最小单位） |
| `W25Qxx_FlashSize` | 0x800000 | 8M 字节总容量 |
| `W25Qxx_FLASH_ID` | 0xEF4017 | W25Q64 JEDEC ID 校验值 |
| `W25Qxx_Mem_Addr` | 0x90000000 | 内存映射模式起始地址 |

### 2.2 通信协议配置

模块在不同阶段采用优化的 SPI 模式：

* 指令层：1 线模式 (Standard SPI) 发送命令。
* 读取层：1-4-4 模式 (Quad-SPI)，使用 6 个空周期 (Dummy Cycles) 实现高速读取。
* 写入层：1-1-4 模式 (Quad-Input Page Program)，提升数据写入带宽。

### 2.3 代码架构

```c
内部辅助函数层：
├─ OSPI_W25Qxx_AutoPollingMemReady() // 硬件自动轮询忙标志位
├─ OSPI_W25Qxx_WriteEnable()         // 发送写使能并校验 WEL 位
├─ OSPI_W25Qxx_ReadID()              // 读取 ID
├─ OSPI_W25Qxx_BlockErase_64K()      // 64KB 块擦除
└─ OSPI_W25Qxx_WritePage()           // 单页写入 (最大 256 字节)

公开 API 层：
├─ Module_W25qxx_init()              // 模块初始化与 ID 校验
├─ Module_W25Qxx_WriteBuffer()       // 自动对齐的跨页数据写入
└─ Module_W25Qxx_ReadBuffer()        // 快速 Quad-IO 数据读取
```

---

## 3. 读写决策逻辑

由于 Flash 物理特性的限制，写入操作遵循以下逻辑：

* 写前擦除：`WriteBuffer` 在执行时默认从地址 0 开始执行 64KB 块擦除，确保写入区域干净。
* 自动分页：当写入数据量超过页边界（256 字节）时，内部逻辑会自动计算剩余空间并切分数据，执行多次页编程指令。
* 硬件轮询：利用 OCTOSPI 的自动轮询功能，在擦除或编程期间不占用 CPU 资源，硬件自动监控 Flash 的 `BUSY` 位。

---

## 5. 注意事项

- 平台限制：目前高速 Quad-SPI 驱动仅在 `STM32H723xx` 平台实现；在 `STM32F407xx` 等平台下接口为空实现，返回错误代码。
- 依赖模块：本模块依赖 `ThreadX (TX_API)` 提供返回状态，及 `HAL_OSPI` 库进行底层通信。
